Java programming

Programming paradigms

  1. Imperative -> How
  2. Declarative -> What

Main functional interfaces

Interface Description
Functions Takes input and return output
Consumers Takes input and no return
Predicates Takes input and return boolean
Suppliers Takes no input and return output
Runnable Represent a task, no input and no output
Callable Represent a task, no input and return output

Refer java.util.function package

Java streams

Way of switch from imperative to declarative programming.

  1. Start with collections [ Arrays, Lists, Sets, Maps ].
  2. Process is done through intermediate operators [ Filter, Map, Reduce etc. ] and returns stream back.
  3. Ends with terminal operators [ Sum, Collect, Average etc. ] and returns a concrete value.

  2. Asynchronous + Blocking
  3. Synchronous + Non-Blocking
  4. Asynchronous + Non-Blocking
