Docker commands usage guide

Overview

  1. Initially, applications were hosted on isolated servers, but this approach was expensive.
  2. Virtualization was introduced, but it suffered from low performance, limiting its popularity.
  3. Docker, introduced by Solomon Hykes, offered a lightweight and efficient solution for application hosting.
  4. Docker is an open-source platform that :
    • packages the application, its dependencies, and runtime together.
    • isolates applications from the underlying host system.
  5. Install Docker

Docker components

  1. Dockerfile: A file without an extension containing instructions to package the application, including its dependencies and runtime environment.
  2. Image: Snapshot of Dockerfile.
  3. Container: A running instance of Image, isolated and lightweight.
  4. Docker Compose: A YAML file (docker-compose.yaml) that serves as a tool to define and manage multiple containers (images) as a single application.
  5. Docker Registry: A repository to store and manage Docker images, allowing easy sharing and distributig.
    • The default public registry is Docker Hub
  6. Workflow that :
    • manages single container
      • Dockerfile -> docker build -> An image -> docker run -> A container
    • manges multiple containers
      • Multiple Dockerfiles and docker-compose.yml -> docker-compose build -> Multiple images -> docker-compose up -> Multiple containers
      • Multiple Dockerfiles and docker-compose.yml -> docker-compose up --build -> Multiple images -> Multiple containers
