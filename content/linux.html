Linux commands usage guide

File operations

  1. List the contents of a directory, including hidden files, with detailed information. ls -la
  2. File copy. cp f1 f2
  3. Rename or move file. mv f1 f2
  4. Remove file rm f
  5. Create an empty file. touch f
  6. Find all txt files in the directory. find d -name ‘*.txt’
  7. Determine the type of a file. file f

Directory operations

  1. Print Working Directory. pwd
  2. Directory copy. cp -r d1 d2
  3. Rename or move director. mv d1 d2
  4. Create a new directory. mkdir d
  5. Remove
    • an empty directory. rmdir d
    • directory and its content. rm -r d
  6. Find all directories starting with 'a' in the directory. find d -name ‘a*’
  7. Change directory cd d
    • to the previous directory. cd -
    • to the parent directory. cd ..
    • to the home directory. cd ~

File viewing

  1. Show content. cat f
  2. Concatenate content of f1 & f2 and display. cat f1 f2
  3. Scroll through the content. less f
  4. Show first 20 lines. head -n 20 f
  5. Show last
    • 20 lines. tail -n 20 f
    • lines as it changes. tail -f f

Access control

  1. Ownership of a file or directory
    • Change owner chown owner-name f
    • Change owner & group chown owner-name:group-name f
  2. Permissions of a file or directory.
    • The owner/user of the file can read, write, and execute the file. The group and others can read and execute the file, but cannot modify it.
      • chmod 755 f [ 7 -> Owner/ User, 5 -> Group, 5 -> Other permissions ]
      • chmod u+rwx,go+rx f
    • Remove write permission from group, remove write and execute from other. chmod g-w,o-wx f
    • Add execute permission for all. chmod a+x f

Redirection operators

  1. Overwriting content if file exists, otherwise file will be created and written. ls > f1.txt
  2. Appending content if file exists, otherwise file will be created and written. ls >> f2.txt
  3. Read the file's content and use it as input for the command. grep 'pattern' < f3.txt
  4. Pass the output of one command as the input to another command. ls | grep 'pattern'

Process management

  1. Process Status
    • Displays a snapshot of currently running processes for all users, excluding daemon processes. ps -a
    • Displays a snapshot of all currently active processes, including background (daemon) processes, for all users. ps -ax
  2. Displays real-time information about active processes, including CPU and memory usage. top
  3. Terminate a process by process id kill pid
    • forcefully kill -9 pid

Networking

  1. View network configuration details for network interfaces ifconfig
  2. Test network connectivity. ping google.com
  3. Transfer data to and from a server :
    • download a file. curl -O https://example.com/f
    • save output to a file. curl https://example.com/f -o f
    • GET request. curl https://example.com
    • POST request. curl -X POST -H "Content-Type: application/json" -d '{"key":"value"}' https://example.com/resource
    • set custom header. curl -H "Authorization: Bearer token" https://example.com
    • view HTTP response headers. curl -I https://example.com
    • trace network activity. curl -v https://example.com
  4. Query DNS :
    • retrieve ip-address of domain dig example.com +short
    • find domain name associated with the ip-address dig -x ip-address
  5. Establish a secure, encrypted connection to a remote host:
    • authentication ssh -i /path/to/private_key user@remote_host
    • via jump host. ssh -J user@jump_host user@target_host
    • forwards a local port to a remote host and port (Tunneling). Maps localhost:8080 on the local machine to localhost:80 on the remote machine. ssh -L 8080:localhost:80 user@remote_host
  6. File transfer. scp /path/to/private_key file-name user@remote_host:/path/to/destination
